企劃背景

創作由來

這是 2019 年我在智慧居家公司任職期間完成的項目，作為市場部經理，同時也負責設計工作。公司正經歷改組，拆分產品研發部門，並改為委外 ODM，同時完成銷售產品線的升級擴充。公司需要在產品圖、產品冊、公眾號與公司網站上一番改版革新。因此這一項目是基於全新的設計元素下共同完成的成果。

上述內容均已完成，只可惜並未實際投入使用太久，公司在項目完成不到三個月就因資金問題而解散。

項目升級

在前端開發的課程結束後，我試著將此一項目升級，由於設計元素共通，我著力在網站的改版上，因為可改善的空間最多。網站的改版大致維持原有的思路，但在細節上調整，並引入當時未完成的回應式設計。

目標

最初想法

網站在開發的最初規劃中，有幾項明確的目標：

  • 僅作為靜態展示的網站，公司主要營運方向還是線下溝通，仰賴銷售人員實際評估後給予方案報價。線上並未配置人力，不做即時互動設計。
  • 相較於前一版本僅是產品的陳列，此版必須增加公司的發展歷程、服務案例與商業合作模式。我自己對於網站的定位是資訊傳播，因此額外建立了頁面說明智慧居家與技術。
  • 產品陳列必須改善前一版本僅有圖片與參數的撰寫方式，需加上應用說明，並盡可能色彩豐富化。
升級目標

在經歷過完整的 UX / UI 與前端開發訓練後，重新檢視此網站，升級將著改善以下內容：

  • 完成回應式設計，針對小螢幕的排版最佳化。
  • 圖標調整，改善圖標風格不統一問題。
  • 翻譯，改由英文呈現。
  • 產品表格呈現調整，省去重要性較低的訊息，對於多色產品，視覺化呈現。

原型

三版塊結構佈局

網站的設計上分為三板塊：Smart Home、Solution 與 About，Smart Home著重於理論面的介紹，不涉及產品與方案，畢竟智慧居家還不是非常成熟且廣為人知的領域，知識面的提供也可以讓網站不那麼功利傾向。

Solution 根據應用場景分類，在逐一展示產品。

About 的部分，除了有公司的基本發展歷程，也列舉了案例強化公司的服務與商業合作能力。

雙導覽列佈局

三板塊結構構成的導覽列還不足以讓網站的導覽更精確，因此加入第二級導覽列。

下拉式的二級導覽列會因為方案內過多拆分元素而變得過長，同時也在參考 Apple 官網的設計後採用兩列式排列，並輔以圖標加深視覺化。

產品呈現精準化

產品的呈現上以正視圖加標籤並排呈現，產品資訊在 Solution 的各頁面中統一整合於下方，並以表格呈現，表格不會承載產品的所有參數保留產品名、材質與顏色。不呈現售價是保留線下銷售的洽談空間，而舉凡尺寸、安裝方式、適用電壓等安裝面內部人員知曉即可。

風格

圖標

本次項目升級很重要一部分是圖標的統一，在最初改版中，Smart Home 與 About 為顏色填充，Solution 的部分延續產品冊的線條方式。此次升級統一為顏色填充，且幾乎重新設計了所有圖標。圖標大量運用圓角，尺寸為 24px。

產品圖

產品圖延續改版初的設計。當時首先嘗試實際拍攝，但考量到不容易統一尺寸，容易照成後續排版困難外，產品拍攝後的精修也非常花時間，有超過 50 樣大小不一的產品。最終參考了一家鋁合金門窗的產品圖測，以向量繪圖的方式呈現。產品在比例、材質、字體、反光與陰影的處理上盡可能擬真。

回應式設計

此次升級的最大目標就是完成回應式設計。大小螢幕在排版上並沒有太大的視覺變化，主要是針對字體大小、間距、表格等做變化。圖片則縮放至最適合大小。

Hover 的大量應用

純靜態的網頁欠缺動態元素，因此增加了大量的 Hover 變化，主要為顏色的變化，不僅在導覽列，內容頁面中許多的圖標與文字連結元素也有應用。

深淺相間的背景配色

頁面的背景色採用深淺交錯的佈局，在產品或圖像較多的頁面採用深色背景，主因是產品多為白色，深色更能凸顯；而在文字較多的頁面則維持白色底，看上去乾淨簡潔。

小螢幕設計呈現

首頁、智慧開關、智慧安防、服務與 Biran 發展歷程頁面。

大螢幕設計呈現

首頁、智慧開關、智慧安防、服務與 Biran 發展歷程頁面。
